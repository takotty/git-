## 💻 程式碼解說

### getRange 四種取得方式

```javascript
// 方法 1：A1 表示法
sheet.getRange("B5")

// 方法 2：A1 範圍
sheet.getRange("A1:D10")

// 方法 3：數字 (列, 欄)
sheet.getRange(5, 2)  // = B5

// 方法 4：數字範圍 (起始列, 起始欄, 列數, 欄數)
sheet.getRange(2, 1, 10, 4)  // A2 開始，10列4欄
```

### 🚨 效能對比：逐格 vs 批次

```javascript
// ❌ 慢！每格一次 API 呼叫
for (var i = 0; i < 100; i++) {
  sheet.getRange(i, 1).setValue(data[i]);  // 100 次 API
}

// ✅ 快！只有 2 次 API 呼叫
var all = sheet.getDataRange().getValues();  // 讀取 1 次
// ... 在記憶體中處理 ...
sheet.getRange(2, 1, result.length, 4).setValues(result);  // 寫入 1 次
```

| 項目 | 逐格讀寫 | 批次讀寫 |
|------|----------|----------|
| API 呼叫次數 | N × M | 2 |
| 速度 | 🐢 慢 | 🚀 快 |
| 記憶體 | 低 | 中 |
| 建議使用 | 單一操作 | 批量操作 |

## ❓ 常見問題 FAQ

### Q：getRange 的列欄從 0 還是 1 開始？
**A**：從 **1** 開始。`getRange(1, 1)` = A1。但 `getValues()` 回傳的陣列索引從 **0** 開始。

### Q：setValues 報錯「尺寸不符」？
**A**：setValues 的陣列維度必須與 Range 大小一致。例如 `getRange(1,1,3,2)` 需要 3×2 的二維陣列。

### Q：處理上千筆資料會超時嗎？
**A**：使用批次讀寫通常不會。但如果超過 6 分鐘執行限制，可考慮分批處理或使用 PropertiesService 記錄進度。

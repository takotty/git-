## 💻 程式碼解說

### SlidesApp 物件層級

```
SlidesApp（服務）
  └─ Presentation（簡報檔案）
       └─ Slide（投影片）
            ├─ Shape / TextBox（形狀/文字方塊）
            │    └─ TextRange（文字內容）
            ├─ Table（表格）
            │    └─ TableCell（儲存格）
            └─ Image（圖片）
```

### 常用 SlidesApp 方法

| 方法 | 說明 |
|------|------|
| `SlidesApp.create(name)` | 建立新簡報 |
| `ppt.getSlides()` | 取得所有投影片 |
| `ppt.appendSlide(layout)` | 新增投影片 |
| `slide.insertTextBox(text, x, y, w, h)` | 插入文字方塊 |
| `slide.insertTable(rows, cols)` | 插入表格 |
| `slide.insertImage(blob)` | 插入圖片 |
| `slide.getBackground().setSolidFill(color)` | 設定背景 |

### 跨服務操作注意事項

| 限制項目 | 說明 |
|----------|------|
| 執行時間 | 6 分鐘上限（跨服務更容易達到） |
| API 配額 | Slides API 每日有呼叫次數限制 |
| 圖表轉移 | 需先轉 Blob 再插入（非直接連結） |
| 權限 | 需要同時授權 Sheets 和 Slides |

## ❓ 常見問題 FAQ

### Q：為什麼 `insertTextBox` 的文字位置不對？
**A**：x, y, w, h 的單位是「點」(points)，1 吋 = 72 點。標準簡報尺寸是 720 × 405 點。

### Q：能不能直接連結試算表圖表到簡報？
**A**：Apps Script 無法建立「連結的圖表」，只能用 `getBlob()` 取得圖表圖片後插入。手動操作可以建立連結圖表。

### Q：一鍵產生簡報太慢怎麼辦？
**A**：跨服務操作（Sheets + Slides）較耗時。可以減少圖表數量或簡化頁面。也要注意 6 分鐘的執行限制。

### Q：如何修改簡報的尺寸（例如 16:9 vs 4:3）？
**A**：`SlidesApp.create()` 預設為 16:9（10×5.625 吋）。目前 Apps Script 無法透過程式修改簡報尺寸，需手動在 Slides 中設定。
